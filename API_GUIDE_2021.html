INTRODUCTION‌

API ADMINISTRATION‌


ONESOURCE API Administration hosts features for administering and managing API application access. The goal is to create a centralized place with simplified user experience to create, read, update and delete your API applications, set rate limits for critical applications, manage 3rd party applications such as Alteryx.



User Administration Service Account Retirement

For API management and use, existing service accounts created under User Administration will be retired and ported over to API Administration as “Applications”. Applications will show details such as Consumer Key, Consumer Secret, etc.


WHO SHOULD READ THIS GUIDE?‌

If you are responsible for the below functionalities:


  • Existing ONESOURCE Administrators role users who are responsible for creating service accounts under User Administration


  • Set up API applications


  • Manage and maintain API Applications including setting Rate Limits


  • User of ONESOURCE Connector for Alteryx, etc.


PREREQUISITES‌

There are no prerequisites required for using API Administration.

HOW TO ACCESS API ADMINISTRATION‌

Accessing to API Administration is controlled by selecting API Administration under ONESOURCE API in Products Permission tab within ONESOURCE Administration. The administrator of the client DB will be the one providing access to users by selecting API Administration.



HOW TO ACCESS API ADMINISTRATION FEATURES‌

Accessing to API Administration features is controlled by permissions set for each user within ONESOURCE Administration. The administrator of the client DB will be the one managing user access by granting the appropriate permissions.



More details on how to set up user permissions can be found in ONESOURCE Administration user guide.


Overview of the API Administration‌

The landing page of the API Administration offers an overview of all applications currently created under the customer account / account ID. This page provides users the capability to create an application, read/edit an application, delete an application and the search/filter capability.




API APPLICATIONS‌


Create an Application‌

Your first step towards API usage is to create an application. To initiate your API journey, click on Add Application.



Once clicked, you will see the following screen. You must choose between User Application or Machine Application to complete your creation.




User Application‌

A user application is an API application that requires an ONESOURCE user to login. One example is the ONESOURCE Connector for Alteryx.


To create an application, input a unique Application Name and Callback URL. The Callback URL is a valid URL that ONESOURCE will be redirected to after credentials authentication. The URL must start with https://.


For Alteryx connector usage, toggle the Alteryx button. Certain fields such as Callbackurl will be hidden from user interface since it is not applicable for Alteryx usage.


Select ONESOURCE products that you have API subscription. Click Done.


Machine Application‌

A machine application is an API application that does not require an ONESOURCE user to login. The application uses a machine-to-machine approach to call ONESOURCE product APIs.


To create an application, input a unique Application Name. Select the type of application authentication.


Client ID and Client Secret - Application is authenticated using a public identifier and a secret according to "OAuth 2.0 Authorization Framework (RFC 6749)"


Client ID and Client Assertion - Application is authenticated using a public identifier and a JSON Web Token called client assertion according to "Assertion Framework for OAuth 2.0 Client Authentication and Authorization Grants (RFC 7521)"


If you select Client ID and Client Assertion, a X.509 certificate is required. User can select New Certificate or Re- use Existing Certificate.



To add a new certificate, add the X.509 certificate.


X.509 certificate - Public key certificate that contains the public key to be used to verify client assertion. Contact your IT team if you are not aware of this certificate.


The certificate must begin with -----BEGIN CERTIFICATE----- and end with -----END CERTIFICATE-----.



To re-use an existing certificate, in the drop-down menu select the issuer of existing certificate.



Select ONESOURCE products that you have API subscription. Click Next.

Next step is to manage Client Setup, Group Setup and Permission Setup.


Client Setup‌

Client Setup is an optional step for client account / account ID which has ONESOURCE Client Manager enabled. This page allows users to appropriately manage specific clients for API access.



When done click Next.


Group Setup‌

Group Setup is an optional step for users to assign groups to provide pre-defined set of permissions.




Users who need to create, modify or delete an existing group may do so in ONESOURCE Administration. When done click Next.

Please note Group permissions changes may take time to come in effect across ONESOURCE Platform.


Permission Setup‌

Permission Setup is the step to set application specific permissions.



Once complete, click Done.

To edit an application, on Applications page click on the edit icon.




When clicked you will see the application details. You may now update information such as Callback URL, Application Authentication and ONESOURCE Products. Once complete, click Next to complete Clients, Group, Permissions tab setting.

Filter Applications‌

You may filter your applications based on column data by clicking on the sort button. The function allows you to quickly filter and display matched applications based on your needs.



Refresh Consumer Key and Secret‌

You may refresh the Consumer Key and Consumer Secret in an application by clicking on the refresh icon. The function will deactivate the existing pair and a new pair will be generated.



When clicked, a pop-up message will be shown and require your confirmation.




Click Confirm to proceed.


CERTIFICATES MANAGEMENT‌

Certificates management offers users a centralized access to manage, create, edit and delete X.509 certificates used for API machine applications. By default all users will have view permission to certificates in use.



Users who need to add, edit and delete a certificate are required to select the following permissions in ONESOURCE Administration. The administrator of the client DB will be the one managing user access by granting the appropriate permissions.




Add a Certificate‌

To add a certificate, click on Add Certificate.



Provide the Issuer and X.509 certificate.


Issuer – The issuer identifies the principal that issued the certificate.


X.509 certificate - Public key certificate that contains the public key to be used to verify client assertion. Contact your IT team if you are not aware of this certificate.


The certificate must begin with -----BEGIN CERTIFICATE----- and end with -----END CERTIFICATE-----.




When done click Done.


View a Certificate‌

To view details of a certificate, on Certificates page click on the Certificate Name.



When clicked, the certificate details will be displayed.




Edit a Certificate‌

To edit a certificate, on Certificates page click on the edit icon.



When clicked you will see the certificate details. You may now update the X.509 certificate. Once complete, click

Done. Note: Issuer cannot be edited.


Delete a Certificate‌

To delete a Certificate, on Certificates page click on the delete icon.



When clicked, ONESOURCE will prompt you to confirm your decision.



Click Delete to proceed.


RATE LIMITS‌

Rate limit is the number of API calls an app can make within a given time period. If this limit is exceeded the app will be throttled and API requests will fail (HTTP status code 429 limit exceeded). All API requests are subject to rate limits. Manage your API rate limits using the below parameters.


Limit - The maximum requests which will be served for that API in a given time period. Once the limit is reached, further requests in that time period will start failing till the end of the time period.


Reserve - The minimum number of requests received for an API which are guaranteed to be served in a given time period. The request number set as a reserve for an API would reduce the maximum requests allowed for other APIs.


View Rate Limits for Applications‌

To view rate limits of your applications, choose the ONESOURCE product that is entitled for API usage.



When selected, a list of all applications assigned with the ONESOURCE product will be displayed.




Edit Rate Limits for Applications‌

To edit rate limits of applications, input a value for Limit or Reserve based on your use case. A single application may have either a Limit or Reserve assigned, but not both. The total percentage of all rate limits cannot exceed 100%. Note that it is not necessary to assign Limits or Reserves to applications.




REPORTS‌


API Usage‌

Users can lookup their API traffic, latency and errors related to customer account / account ID.




Create a Report

To create a report, click on Create Report.



Input a unique report name, select the date range, time unit and ONESOURCE products.




When complete, click Create. The system will start preparing for the report and it may take some time depends on amount of data processed. When the report is ready, a download icon will be displayed.




Users can click on the download icon to save a .csv file for immediate use. The report will display the following details related to API usage. Note: each report created has a download expiration. Users should download the report at their earliest convenience.



Search a Report

To search a report, on API Usage page use the search box and type in your report name.



Delete a Report

Users can delete a report by clicking on the delete icon.



API Application History‌

Users can look up details of change made to applications and permissions.



Search an Application History

To search an application history or permission change, input the mandatory date range. Start Date cannot be a future date and cannot be after End Date. Similarly, End Date cannot be a future date. Users can put in optional search parameters such as Application Name, Application Type, Activity Type and ONESOURCE Products to restrict search result.




When done, click Submit.


A list of results found based on search criteria will be displayed with most recent activity date at the top.




To view details of each entry, click on an entry and a detailed page will pop up on the right.



Highlighted fields represent values have changed compared to previous state.




To download the report, click on Download button.



A .csv file will be downloaded to your local drive. Note that each change history corresponds to one line entry.
