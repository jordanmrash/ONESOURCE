Best Practices – Working with Scenarios


  • Best Practices – Working with Scenarios –Entity Combine

    • During an Entity Combine, the liquidated entities are essentially converted into divisions of the sole survivor

    • The following operations are performed

      –Liquidated entities are converted to divisions of the survivor

      –Any divisions of liquidated entities become divisions of the survivor as well

      –Survivor also gains any filings, nexus, and group memberships of the liquidated entities


      Best Practices – Working with Scenarios


  • Best Practices – Working with Scenarios – Entity Split

    • Split from an existing entity to create a new entity

      • The new entity can be a Division, FTE, or Separate Entity

    • The new entity will have filings in the selected jurisdictions, and inherit nexus from the original entity

    • The following operations are performed:

      • Subtract specified amounts from the original entity and add those amounts to the corresponding lines for the new entity

      • Create specified nexus and factors for the new entity

      • Make the new entity a member of all groups where the original entity is a member

OHIO CITIES INTEGRATION

The transfer of the Ohio cities’ apportionment data is not supported by the direct integration between ONESOURCE State Apportionment (OSA) and ONESOURCE Income Tax (OIT). As an alternative, the apportionment for these jurisdictions can be transferred using export/import functionality. This document walks thru the steps to set up and run this process.



ONESOURCE State Apportionment – Export Configuration and .dif file creation


Create Export Jurisdiction Map

  1. Create a jurisdiction map to assign OSA Ohio cities’ jurisdiction IDs to the corresponding 3 character OIT jurisdiction ID.

  2. Create the map using whatever name you like.

  3. Press the Default button to assign initial mappings.

  4. Click the checkbox in the header section to deselect all jurisdictions.

  5. Select the Ohio city jurisdictions you wish to export and set the export ID to OIT’s three character code. The list of codes can be found in the .zip file that contained this User Guide. Any jurisdictions that are not mapped to a valid code will be ignored.

  6. Save the jurisdiction map.


  7. *Steps 2 - 4 can also be done by exporting the jurisdiction map, making changes, and importing back the updated file. Please see the OSA user guide for details.


Create Export Account Map

  1. An account map must be assigned to all export templates.

  2. Create an export account map using whatever name you like.

  3. Press the Default button to assign initial mappings. No other settings need be changed.

  4. Save the account map.



Edit Export Template

  1. The export template is called Income Tax – OH Cities.

  2. Create the export template (if it does not already exist)

    1. Navigate: Export Tab -> Export Templates and click <Add>

    2. Enter the name as it appears here: Income Tax – OH Cities

    3. Data Type: Account Data

    4. Export/BOY/EOY Values: Separate Columns for the Same Year

    5. Click Add






  1. Confirm template settings, and assign mappings.

    1. Options:

      1. Category: ONESOURCE

      2. File Format: CSV

      3. Active: Checked

      4. Select File Option: One File For Each Entity

    2. Mappings:

      1. Entity Map: None

      2. Jurisdiction Map: Select the Jurisdiction Map created above

      3. Account Map: Select the Account Map created above.

      4. Elim Map: None

    3. Data Options:

      1. Export Data for:

        1. Entities: checked

        2. Divisions: checked

        3. Groups: unchecked

      2. Logic Rules

        1. Include Zero Balances: Unchecked

        2. Don’t Aggregate Data: Unchecked

        3. Rollup Division to Parent: Optional. Generally this box should be checked. It should be unchecked only if the divisions have their own binders in OIT.

        4. Include FTE Flows: Checked

        5. Keep SMLLC Flow Separate: Optional. Generally this box should be unchecked. It should be checked only if the SMLLCs have their own binders in OIT.

        6. Flow all FTEs: Unchecked

        7. Capitalize Rent: Unchecked

        8. Apply Account Tax Rules: Checked


  1. Columns:

    1. Click the Add Column button to create seven columns

    2. Assign Fields to each column in this order

      1. Tax Year

      2. Entity ID

      3. Jurisdiction ID

      4. Factor

      5. Account ID

      6. BOY In-State Amount

      7. EOY In-State Amount

e. Click Save

  5. When prompted, Save the .zip file to your computer. The file will contain one .dif file for each entity you selected to import. Unzip the file and save the .dif files to the desired location.

    Working With Scenarios


  • Planning Wizards –Entity Combine

    • During an Entity Combine, the liquidated entities are essentially converted into divisions of the sole survivor

    • The following operations are performed

      • Liquidated entities are converted to divisions of the survivor

      • Any divisions of liquidated entities become divisions of the survivor as well

      • Survivor also gains any filings, nexus, and group memberships of the liquidated entities


        Working With Scenarios


  • Planning Wizards –Entity Split

    • Split from an existing entity to create a new entity

      • The new entity can be a Division, FTE, or Separate Entity

    • The new entity will have filings in the selected jurisdictions, and inherit nexus from the original entity

    • The following operations are performed:

      • Subtract specified amounts from the original entity and add those amounts to the corresponding lines for the new entity

      • Create specified nexus and factors for the new entity

      • Make the new entity a member of all groups where the original entity is a member


        Working With Scenarios
